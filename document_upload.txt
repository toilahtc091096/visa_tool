Có, nhưng **không thể đính kèm trực tiếp một thư mục (folder) vào email** theo chuẩn SMTP/MIME. Bạn có một số lựa chọn:

### Cách 1: Nén folder thành file ZIP (phổ biến nhất)

Giả sử cấu trúc:

```
documents/
├── A/
│   ├── a1.pdf
│   └── a2.pdf
├── B/
│   └── b1.pdf
└── C/
    └── c1.pdf
```

Bạn nén:

```
documents.zip
```

và gửi `documents.zip` làm file đính kèm.

Python:

```python
import shutil

shutil.make_archive(
    "documents",   # tên file zip
    "zip",
    "documents"    # folder cần nén
)
```

Sau đó đính kèm:

```python
attachments = ["documents.zip"]
```

---

### Cách 2: Đính kèm toàn bộ PDF trong folder và folder con

Nếu không muốn nén, bạn có thể duyệt đệ quy và đính kèm từng PDF.

```python
from pathlib import Path

attachments = []

for pdf in Path("documents").rglob("*.pdf"):
    attachments.append(pdf)
```

hoặc

```python
for file in attachments:
    print(file)
```

Kết quả:

```
documents/A/a1.pdf
documents/A/a2.pdf
documents/B/b1.pdf
documents/C/c1.pdf
```

Sau đó lần lượt attach từng file.

---

### Cách 3: Giữ nguyên cấu trúc thư mục

Nếu muốn người nhận vẫn thấy cấu trúc:

```
documents/
    A/
    B/
    C/
```

thì **bắt buộc phải nén (ZIP/RAR/7z)**. Chuẩn email không hỗ trợ gửi một cây thư mục.

---

## Khuyến nghị

* Nếu chỉ có vài file PDF → đính kèm từng file.
* Nếu có nhiều folder con và muốn giữ nguyên cấu trúc → nén thành ZIP rồi gửi.
* Nếu tổng dung lượng lớn (20–25 MB trở lên, tùy nhà cung cấp email) → nên tải lên dịch vụ lưu trữ (Google Drive, OneDrive, Dropbox...) và gửi liên kết thay vì đính kèm trực tiếp.
